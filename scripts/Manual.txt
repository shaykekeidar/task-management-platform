Task Generator - Deployment Manual
Architecture

This project uses:

Docker for application packaging
Helm for Kubernetes deployments
PowerShell for CI/CD automation
Kubernetes namespaces for environment separation

Namespaces:

task-generator-dev
task-generator-test

Applications:

frontend
users
auth
tasks
Version Management

Application versions are stored in:

app\application-versions.txt

Example:

users=1
auth=1
tasks=1
frontend=11

Before releasing a new version:

Update the relevant version number.
Save the file.
Run the appropriate deployment script.

Script Overview

################## buildImage.ps1 ##################

Purpose:

Builds and pushes a Docker image.

Flow:

Reads version from application-versions.txt
Builds Docker image
Pushes image to Docker Hub

Example:

usage: auth, frontend, tasks, users

.\buildImage.ps1 frontend

Result:

shaykekeidar/kub-demo-frontend:11

Use when:

You only want to build and push an image.
You do not want to deploy yet.

################## deployHelm.ps1 ##################

Purpose:

Deploys a specific version using Helm.

Flow:

Select environment
Override image tag using Helm --set
Execute helm upgrade --install

Examples:

.\deployHelm.ps1 frontend dev 11

.\deployHelm.ps1 frontend test 11

Use when:

Image already exists in Docker Hub.
You only want to deploy.



##################  release.ps1 ##################

Purpose:

Complete CI/CD flow.

Flow:

Build image
Push image
Deploy with Helm

Examples:

.\release.ps1 frontend dev

.\release.ps1 frontend test

Use when:

Releasing a new version.
Normal day-to-day deployment.

This is the recommended script.
